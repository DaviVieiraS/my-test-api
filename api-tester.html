Vercel Setup Instructions

Backend Setup on Vercel

  1. Create a new directory for your project
  2. Inside, create an api folder
  3. Create two files: user.js and product.js in the api folder
  4. Deploy to Vercel by running vercel in your terminal

user.js (GET endpoint)

export default function handler(request, response) {
  const { username } = request.query;
  
  // Example user data - replace with your own logic
  const users = {
    john_doe: { name: "John Doe", email: "john@example.com", role: "admin" },
    jane_smith: { name: "Jane Smith", email: "jane@example.com", role: "user" }
  };
  
  if (users[username]) {
    response.status(200).json(users[username]);
  } else {
    response.status(404).json({ error: "User not found" });
  }
}

product.js (POST endpoint)

export default function handler(request, response) {
  if (request.method === 'POST') {
    const { username, productId, status } = request.body;
    
    // Here you would typically save to a database
    // This is just a simulation
    const success = Math.random() > 0.2; // 80% success rate for demo
    
    if (success) {
      response.status(200).json({ 
        message: "Product status updated successfully",
        productId,
        status,
        updatedBy: username
      });
    } else {
      response.status(500).json({ error: "Failed to update product status" });
    }
  } else {
    response.status(405).json({ error: "Method not allowed" });
  }
}

Frontend Usage

Once deployed, update the endpoint URLs in the form to match your Vercel deployment.

For example, if your Vercel domain is https://my-api.vercel.app, set your endpoints to:

  • GET: https://my-api.vercel.app/api/user
  • POST: https://my-api.vercel.app/api/product
